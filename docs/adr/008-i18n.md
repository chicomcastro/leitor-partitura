# ADR 008 — Internationalization (i18n)

## Status
Accepted

## Context
The app targets musicians globally but was initially Portuguese-only. Full i18n libraries (react-intl, i18next) add significant bundle size for a simple use case.

## Decision
Custom lightweight i18n using React Context. Translation strings stored in a single `src/lib/i18n.jsx` file. Locale persisted in `localStorage('sp.locale')`. Default locale is `pt-BR`, with `en` as the second language. Language toggle in the Library header.

## Consequences
- Zero additional dependencies, ~3KB added to bundle
- Adding a new locale requires only adding a translation object to the file
- No ICU message format or pluralization rules — simple key-value lookup suffices for this UI
- Context-based approach ensures all components re-render on locale change
